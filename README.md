# 📄 PDF Comparison Project

> ระบบแปลงและเปรียบเทียบเอกสาร PDF อัตโนมัติ (Side-by-side) ช่วยเปรียบเทียบหน้าแรกของเอกสารต้นฉบับ (Original) และเอกสารฉบับแก้ไข (Revised) ได้อย่างรวดเร็วและแม่นยำ

## ✨ คุณสมบัติเด่น (Features)

- 🔍 **จับคู่อัตโนมัติ (Regex Parsing):** ค้นหาและจับคู่ PDF ผ่านรหัส 3 หลักแรกของชื่อไฟล์ (`^(\d{3})`)
- ⚡ **ประมวลผลคู่ขนาน (Parallel Execution):** เร่งความเร็วการแปลงและผสานภาพด้วย `ThreadPoolExecutor` ช่วยจัดการไฟล์ปริมาณมากได้อย่างรวดเร็ว
- 🖼️ **มุมมองเปรียบเทียบ (Side-by-Side):** แปลงเนื้อหา PDF เฉพาะหน้าแรกที่ 150 DPI เพื่อนำมาต่อกันทางแนวนอน (ซ้าย-ขวา) บนพื้นหลังสีขาว
- 📄 **สร้างเอกสารรายงานอัตโนมัติ:**
  - 📝 **`docx` (MS Word):** จัดรูปแบบกระดาษเป็น **แนวนอน (Landscape)** และพยายามบีบ Margin ให้แคบเพื่อจำลองการแสดงผลภาพที่สมบูรณ์ที่สุดที่ความกว้าง 9.5 นิ้ว
  - 📑 **`pdf` (Multi-page PDF):** รวบรวมข้อมูลจากการเปรียบเทียบเป็นแฟ้มภาพหลายหน้าด้วยฟีเจอร์ append ของไลบรารี Pillow
- 🖱️ **User-Friendly GUI:** แสดงหน้าต่างแบบรวบรัด (Native Tkinter `askdirectory`) เพื่อให้ผู้ใช้ชี้เป้าไฟล์ได้ง่ายที่สุดโดยไม่ต้องพิมพ์ Path เอาเอง

## 📂 โครงสร้างโปรเจกต์ (Project Architecture)

โปรเจกต์นี้ถูกออกแบบมาเพื่อแยกกระบวนการทำงานที่เป็นตรรกะ (Logic) การจัดการไฟล์เข้าออก (I/O) เข้าไว้เป็นโมดูล (Modular):

```text
pdf_comparison_project/
├── data/
│   ├── original_docs/   # 📥 โฟลเดอร์ (ตัวอย่าง) พักไฟล์ต้นฉบับ
│   └── revised_docs/    # 📥 โฟลเดอร์ (ตัวอย่าง) พักไฟล์หลังการแก้ไข
├── output/              # 📤 รายงานผลลัพธ์จะถูกสร้างและเก็บในโฟลเดอร์นี้
│   ├── Comparison_Report.docx
│   └── Comparison_Report.pdf
├── src/
│   ├── core/            # 🧠 ส่วนประมวลผลตรรกะ (Logic & Processing)
│   │   ├── comparison.py    # จัดการ Flow การทำงานหลัก
│   │   ├── matcher.py       # ดึงรายชื่อไฟล์และค้นหาคู่อย่างปลอดภัย
│   │   ├── rasterizer.py    # แปลงไฟล์ PDF เป็นภาพผ่าน Byte Buffer
│   │   ├── concatenator.py  # ตัดแต่งและนำภาพเข้าคู่ ผสานในรูปแบบซ้าย-ขวา
│   │   └── generator.py     # Export ผลลัพธ์กลับสู่ DOCX และ PDF
│   └── ui/              # 🖥️ ส่วนประกอบหน้าตาหน้าต่างแอปพลิเคชัน (GUI)
│       └── app.py           # หน้าต่างแบบสอบถาม CustomTkinter
├── run.py               # 🚀 File หลักสำหรับเรียกใช้งานโปรแกรม (Entry Point)
├── .gitignore           # กฎละเว้นไฟล์สำหรับ Git
├── README.md            # คู่มือการติดตั้งและการใช้งานโครงการนี้ (คุณกำลังอ่านไฟล์นี้อยู่)
└── requirements.txt     # Dependency ทั้งหมด (PyMuPDF, Pillow, python-docx, customtkinter)
```

## 🛠️ การติดตั้ง (Installation)

**1. Clone โปรเจกต์ลงเครื่องของคุณ:**
```bash
git clone <your-repository-url>
cd pdf_comparison_project
```

**2. ติดตั้งไลบรารีที่จำเป็นผ่าน pip:**
```bash
pip install -r requirements.txt
```

> [!TIP] 
> แนะนำให้ใช้งานใน Environment แยก (เช่น `venv` หรือ `conda`) เพื่อลดปัญหาไลบรารีชนกัน

## 🎯 วิธีการใช้งาน (Usage Guide)

1. **เตรียมไฟล์ (Data Preparation)**
   - เปลี่ยนชื่อเอกสารที่ต้องการเปรียบเทียบ โดยใช้หลักการใส่ **รหัสตัวเลขสามหลักนำหน้า** เหมือนกัน เช่น:
     - ต้นฉบับ: `001_contract_v1.pdf` โฟลเดอร์ `Original`
     - แก้ไข: `001_draft_v2.pdf` โฟลเดอร์ `Revised`
   - ระบบอนุญาตให้มีคำต่อท้ายอะไรก็ได้หลัง 3 หลักแรก

2. **สั่งรันโปรแกรม (Execution)**
   เปิด Terminal ของคุณและใช้คำสั่งจาก Root Folder:
   ```bash
   python run.py
   ```

3. **ดำเนินการผ่าน GUI**
   - โปรแกรมจะแสดงหน้าต่างให้ขึ้นมาเพื่อเลือก Directory อย่างเรียบง่าย
   - เลือก **โฟลเดอร์สำหรับเอกสารต้นฉบับ**
   - เลือก **โฟลเดอร์สำหรับเอกสารฉบับแก้ไข**

4. **ตรวจสอบผลลัพธ์ (Output Analysis)** 
   ระบบจะแสดงความคืบหน้า (Progress) เป็นเปอร์เซ็นต์ หลังจากการทำงานคำนวณ 100% เรียบร้อยแล้ว สามารถเข้าดูผลที่รวมแล้วทุกรายการได้ที่โฟลเดอร์ `output/`

---

> [!WARNING]
> **ข้อจำกัดของระบบที่ควรทราบ:**
> - ระบบนี้ทำการ Render **เฉพาะหน้าแรก (Page 0)** ของ PDF เท่านั้น เพื่อลดระยะเวลาการทำงานและป้องกันหน่วยความจำล้น (Memory Overflow) 
> - ปัญหาการแครชของรายงาน Word บางประการได้ทำการแก้ไขผ่านการตั้งค่า Spacing `Pt(0)` เรียบร้อยแล้ว (ไม่พบหน้าว่าง)
> - ไม่ควรมีไฟล์ PDF ที่ตั้งรหัสตัวเลข 3 หลักซ้ำกันในโฟลเดอร์เดียวกัน ป้องกันการจับคู่ทับซ้อน (Matching Collision)
